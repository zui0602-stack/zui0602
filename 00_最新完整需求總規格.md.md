## 2. 最終目標

製作一套萬龍覺醒自動化腳本，核心是：

```text
角色管理總控
→ 掃描角色
→ 統一登入角色
→ 登入後先處理主城資源與聯盟相關
→ 執行採集或每日任務
→ 讀取部隊倒數
→ 回到角色管理介面等待
→ 倒數或指定時間到後再登入對應角色
→ 持續循環
```

腳本最終不是只做單角色採集，而是完整多角色總控。

---

## 3. 最重要原則

1. 最終必須停留在「角色管理」介面等待。
2. 角色管理介面必須顯示目前已偵測角色的倒數或等待狀態。
3. 偵測到幾個角色，就顯示幾個角色，不固定顯示 5 個。
4. 所有登入方式都從「角色管理」統一進行。
5. 採集、每日任務、補漏檢查，都共用同一套角色登入流程。
6. 每一步動作前後都必須偵測畫面。
7. 畫面不對時不能硬點、不能跳步、不能亂跑下一步。
8. 所有等待必須有超時。
9. 所有步驟必須有成功判斷與失敗重試。
10. 圖片/OCR 都要限定區域，不可長時間全屏 OCR。
11. 滑動期間不識別，滑動停穩後等待 500～1000ms 再 OCR/找圖。
12. 不可以只靠固定延時。
13. 不可以做成一條超大流程，必須模組化。
14. 缺圖片素材時，要列出需要補的素材清單。
15. 卡住時停止亂點，通知使用者處理。

## 5. UI 介面需求

### 5.1 總開關

```text
[ ] 啟用採集總控
[ ] 啟用每日任務
[ ] 啟用每日補漏檢查
[ ] 啟用防卡通知
```

---

### 5.2 採集時間設定

採集時間必須有獨立開關，避免 UI 裡原本有輸入時間，但使用者其實不想啟用時間限制時，腳本誤判不採集。

```text
[ ] 啟用採集時間限制
採集開始時間：[00:00]
採集結束時間：[23:59]
```

規則：

```text
若「啟用採集時間限制」關閉：
    只要「啟用採集總控」開啟，就允許採集，不看開始/結束時間。

若「啟用採集時間限制」開啟：
    只有目前時間在採集開始～採集結束範圍內，才允許採集。

若開始時間小於結束時間：
    例如 08:00～23:59，代表同一天時間區間。

若開始時間大於結束時間：
    例如 18:50～07:59，代表跨日時間區間。

若時間格式錯誤：
    腳本不可亂跑，必須提示設定錯誤並停止或回角色管理等待。
```

---

### 5.3 採集時間外顯示規則

若「啟用採集時間限制」已開啟，且目前不在採集時間內：

```text
1. 腳本必須停在角色管理介面。
2. 不登入角色採集。
3. 不使用舊檔案紀錄的倒數時間去觸發採集。
4. 不因舊倒數到期而登入角色。
5. 角色管理等待畫面要顯示明確狀態：

   啟用採集時間未到，等待中

6. 若已掃描到角色，可顯示角色列表，但不得因舊倒數觸發登入。
7. 等待期間只允許檢查：
   - 每日正式任務時間是否到
   - 每日補漏檢查時間是否到
   - 採集開始時間是否到

8. 採集開始時間到後：
   - 重新掃描角色管理角色。
   - 重新建立本輪角色倒數狀態。
   - 不直接相信舊檔案倒數。
   - 再開始採集總控。
```

---

### 5.4 每日任務時間設定

每日任務每天只做兩次：

1. 正式執行一次。
2. 補漏檢查一次。

UI 需要包含：

```text
每日正式執行時間：[08:00]
每日補漏檢查時間：[20:00]
```

規則：

```text
每日正式任務：
    每天固定時間執行一次。
    執行完成後，今日不再重複正式執行。

每日補漏檢查：
    每天固定時間執行一次。
    主要檢查是否有每日任務漏做、漏領。
    補漏完成後，今日不再重複補漏檢查。

每日任務優先權高於採集。
每日任務時間到，採集必須暫停。
每日任務完成後，回角色管理等待。
若啟用採集時間限制，必須等到採集開始時間才開始採集。
```

---

### 5.5 提前登入準備時間

提前登入準備時間使用秒數，不只用分鐘。

```text
提前登入準備秒數：[180]
```

規則：

```text
角色倒數剩餘時間 <= 提前登入準備秒數：
    允許登入該角色準備採集。

例如：
    角色 001 倒數 180 秒
    提前登入準備秒數 180
    腳本可以登入 001。

若設定為 0：
    代表倒數到 0 才登入。
```

---

### 5.6 角色設定

```text
角色模式：
[ ] 自動偵測星標角色
[ ] 只登入指定角色

指定角色：
[ ] 角色 1
[ ] 角色 2
[ ] 角色 3
[ ] 角色 4
[ ] 角色 5
```

規則：

```text
最多支援 5 個角色。
角色數量不固定，可能 1～5 個。
偵測到幾個角色，就顯示幾個角色倒數。
被鎖角色、不需要登入的角色，不加星標。
```

---

### 5.7 採集設定

```text
資源策略：
    平衡
    隨機
    固定金
    固定木
    固定石

最高資源等級：[10]
最低資源等級：[1]
最大派遣嘗試：[10]
```

規則：

```text
資源類型：
    金、木、石。

資源策略：
    平衡：盡量讓金木石輪流採集。
    隨機：每次隨機選金木石。
    固定金：只採金。
    固定木：只採木。
    固定石：只採石。

資源等級：
    從最高等級往最低等級搜尋。
    若最高等級找不到，就降一級繼續找。
    直到最低等級。
```

## 6. 角色管理掃描需求

### 6.1 掃描入口

腳本必須能進入：

```text
左上角頭像
→ 設定
→ 角色管理
```

或從其他畫面返回角色管理。

---

### 6.2 掃描方式

角色管理最多 5 個角色，但畫面可能無法一次看到全部，所以必須上下滑動掃描。

```text
進入角色管理後：
    先掃描當前畫面。
    向下滑動一次，等待畫面停穩，再掃描。
    再向下滑動第二次，等待畫面停穩，再掃描。
    再向上滑動一次，等待畫面停穩，再掃描。
    再向上滑動第二次，等待畫面停穩，再掃描。

總共上下滑動兩次，以免少掃角色。
滑動期間不做 OCR/找圖。
每次滑動停止後等待 500～1000ms 再偵測。
```

---

### 6.3 掃描內容

需要偵測：

```text
角色名稱 / 角色 ID
星標狀態
綠色勾勾狀態
角色所在畫面位置
```

規則：

```text
星標角色：
    代表需要登入的角色。

指定角色：
    依 UI 勾選登入指定角色。

綠色勾勾：
    代表目前已登入角色。
```

---

### 6.4 去重規則

因為上下滑動會重複掃到同一個角色，所以必須去重。

去重依據優先順序：

```text
1. 角色 ID / 名稱 OCR 結果
2. 角色所在行特徵
3. 星標 + 名稱區域
```

同一角色不可重複加入角色清單。

## 7. 統一登入流程

所有功能都使用同一套登入流程，包括：

```text
採集
每日正式任務
每日補漏檢查
主城資源
聯盟相關
```

---

### 7.1 一般角色登入

流程：

```text
角色管理
→ 找到指定角色行
→ 點擊角色
→ 若出現「確定 / 取消」視窗
→ 點擊「確定」
→ 等待登入載入
→ 判斷登入完成
```

登入完成判斷：

```text
出現左上角大頭貼
或
出現「出城」按鈕
```

任一條件成立，即視為登入完成。

---

### 7.2 綠色勾勾角色登入特殊規則

角色管理介面上，如果某個角色行有綠色勾勾，代表目前已經登入該角色。

假設綠色勾勾在第一個角色，而腳本下一步剛好要登入第一個角色：

```text
點擊該角色後，不會出現「確定 / 取消」視窗。
```

因此不能一直等待確認視窗。

正確處理方式：

```text
若目標角色行有綠色勾勾：
    點擊該角色或確認目前角色已登入。
    不等待「確定 / 取消」視窗。
    使用返回鍵返回。
    重複返回，直到回到主城畫面。
    以左上角大頭貼或出城按鈕判斷已回到主城。
```

返回規則：

```text
每按一次返回後等待畫面穩定。
檢查是否出現主城特徵。
若出現左上角大頭貼或出城按鈕，停止返回。
若超過最大返回次數仍未回主城，判定登入流程失敗，進入防卡處理。
```

---

### 7.3 登入失敗處理

登入失敗情況包括：

```text
找不到角色。
點擊角色無反應。
需要確認登入時，沒有出現「確定 / 取消」視窗。
非綠色勾勾角色登入後載入過久。
綠色勾勾角色按返回後，無法回到主城。
未出現左上角大頭貼。
未出現出城按鈕。
出現角色被封、斷線、異常彈窗。
畫面停在未知狀態。
```

處理方式：

```text
1. 單一角色登入最多重試 3 次。
2. 每次重試都要重新判斷目前畫面狀態。
3. 若是綠色勾勾目前角色：
    - 不等待「確定 / 取消」視窗。
    - 使用返回鍵嘗試回主城。
    - 每次返回後檢查左上角大頭貼或出城按鈕。
4. 若是非綠色勾勾角色：
    - 點擊角色後等待「確定 / 取消」視窗。
    - 有「確定」就點確定。
    - 沒有視窗就重試。
5. 重試 3 次仍失敗：
    - 立即通知使用者。
    - 通知內容要包含：
        - 角色 ID / 角色名稱
        - 失敗原因
        - 目前所在畫面
        - 已重試次數
    - 該角色標記為登入失敗。
    - 不再無限重試。
6. 若還有其他角色：
    - 可跳過該角色，繼續下一個角色。
7. 若所有角色都登入失敗：
    - 停在角色管理。
    - 顯示「所有角色登入失敗，等待人工處理」。
    - 發送通知。
```

---

### 7.4 登入失敗通知格式

只要角色登入連續失敗 3 次，就必須通知使用者。

通知格式：

```text
角色登入失敗
角色：001
原因：找不到出城按鈕 / 無法回主城 / 登入確認視窗未出現
重試：3 次
狀態：已跳過此角色或等待人工處理
```

## 8. 採集前固定流程

每次角色登入完成後，在採集前固定先執行以下內容。

不需要獨立分鐘間隔。

```text
登入完成
→ 主城資源收取
→ 聯盟領土資源
→ 聯盟寶箱
→ 科技捐獻
→ 再開始採集
```

---

### 8.1 主城資源收取

```text
每次角色登入完成後都檢查主城產量資源。
若資源可收，執行收取。
若沒有資源可收，跳過。
不可卡在這裡太久。
每一步必須有超時。
```

---

### 8.2 聯盟相關

每次角色登入完成後都檢查：

```text
聯盟領土資源
聯盟寶箱
科技捐獻
```

規則：

```text
聯盟領土資源：
    有可領就領。
    沒有可領就跳過。

聯盟寶箱：
    有可領就領。
    沒有可領就跳過。

科技捐獻：
    只使用藍色資源。
    不使用黃色寶石。
    若只剩寶石捐獻，立即停止科技捐獻並返回。
```

## 9. 採集流程

### 9.1 採集主流程

```text
登入完成
→ 採集前固定流程
→ 點「出城」
→ 進入野外
→ 點放大鏡
→ 根據 UI 的「資源策略」決定本輪資源類型
→ 選資源類型：金 / 木 / 石
→ 選資源等級
→ 從最高等級往最低等級搜尋
→ 點搜尋
→ 畫面跳轉到資源點
→ 點資源點
→ 點「採集」
→ 判斷是否出現「創建部隊」
```

資源策略必須套用 UI 設定：

```text
平衡：
    依角色或派遣次數平均分配金 / 木 / 石。
    不可長時間只採同一種資源。
    建議紀錄每個角色最近採集過的資源類型，下一輪優先補少的資源。

隨機：
    每次派遣前隨機選擇金 / 木 / 石。
    若目前資源類型找不到資源點，切換其他資源。

固定金：
    只採金。
    若金找不到資源點，依設定可降低等級繼續找。

固定木：
    只採木。
    若木找不到資源點，依設定可降低等級繼續找。

固定石：
    只採石。
    若石找不到資源點，依設定可降低等級繼續找。
```

找不到資源時：

```text
若目前資源類型與等級找不到資源點：
    先降低等級。
    若降到最低等級仍找不到：
        平衡 / 隨機模式：切換下一種資源。
        固定模式：記錄此角色本輪找不到該固定資源，停止或稍後重試。
```

---

### 9.2 創建部隊判斷

停止條件不能以 5/5 為主。

主停止條件是：

```text
點「採集」後，有沒有出現「創建部隊」。
```

規則：

```text
若出現「創建部隊」：
    點擊創建部隊
    進入編隊流程

若沒有出現「創建部隊」：
    判定該角色部隊已派滿
    停止派遣
    進入部隊管理讀倒數
```

---

### 9.3 編隊與行軍流程

```text
創建部隊
→ 偵測副將紅點 / 副將位置
→ 取消副將
→ 保留預設主將
→ 點擊行軍
→ 等待行軍成功
→ 回到野外
→ 繼續下一輪採集
```

規則：

```text
必須取消副將小紅點。
使用默認英雄進行採集。
不可因副將紅點卡住。
行軍後必須判斷是否回到野外或出現放大鏡。
```

---

### 9.4 找不到資源點

若某一資源/等級找不到資源點：

```text
先降低等級繼續搜尋。
若低到最低等級仍找不到：
    切換下一種資源。
若所有資源與等級都找不到：
    記錄本角色採集失敗。
    回角色管理或進入防卡處理。
```

## 10. 部隊管理與倒數紀錄

### 10.1 進入部隊管理

該角色無法創建部隊後：

```text
打開部隊管理
→ 檢查所有部隊狀態
```

需要偵測狀態：

```text
採集中
行軍中
返回中
```

---

### 10.2 等待所有部隊進入採集中

```text
若部隊仍在行軍中：
    等待，直到變成採集中。

若部隊返回中：
    代表該部隊可能已快完成或回城。
    視情況重新派遣或等待狀態更新。

若所有派出的部隊都顯示採集中：
    讀取倒數時間。
```

必須有最大等待時間，避免卡死。

---

### 10.3 倒數紀錄方式

每個角色不記錄所有部隊時間總和，改記錄「最快完成的一隊」。

原因：

```text
總控需要知道哪個角色最早需要重新登入採集。
所以應該記錄該角色所有部隊中最短倒數。
```

範例：

```text
角色 001：
部隊 1：01:55:30
部隊 2：02:01:10
部隊 3：01:48:20

紀錄：
角色 001：01:48:20
```

---

### 10.4 倒數顯示

角色管理介面等待時，顯示：

```text
角色倒數總控

001：01:48:20
002：00:36:10
003：02:03:40
```

規則：

```text
偵測到幾個角色，就顯示幾個角色。
不顯示未偵測到的空角色。
若某角色尚未取得倒數，顯示「未紀錄」或「等待派遣」。
若採集時間未到，顯示「啟用採集時間未到，等待中」。
採集時間未到時，不可用舊檔倒數觸發登入。
```

## 11. 角色管理等待總控

所有角色採集完成後，腳本必須回到角色管理介面等待。

主等待邏輯：

```text
停在角色管理
→ 顯示每個角色倒數或等待狀態
→ 判斷每日正式任務時間
→ 判斷每日補漏檢查時間
→ 判斷採集時間限制是否開啟
→ 判斷哪個角色倒數最早到
→ 到時間後登入該角色
```

---

### 11.1 採集時間內

若啟用採集總控，且採集時間允許：

```text
找到倒數最短角色。
若該角色剩餘時間 <= 提前登入準備秒數：
    登入該角色。
    執行採集前固定流程。
    執行採集。
    讀取新倒數。
    回角色管理。
```

---

### 11.2 採集時間外

若啟用採集時間限制，且目前不在採集時間內：

```text
停在角色管理。
顯示：啟用採集時間未到，等待中。
不登入採集。
不使用舊檔案紀錄的倒數時間觸發採集。
等待每日任務時間、每日補漏時間或採集開始時間。
```

採集開始時間到後：

```text
重新掃描角色。
重新建立本輪角色倒數狀態。
再開始採集總控。
```

## 12. 每日任務流程

每日任務登入方式與採集完全一樣，必須從角色管理開始。

---

### 12.1 每日正式任務

```text
到每日正式執行時間
→ 暫停採集
→ 回角色管理
→ 掃描角色
→ 登入第一個角色
→ 判斷登入完成
→ 執行每日任務完整流程
→ 執行主城資源與聯盟相關
→ 回角色管理
→ 登入第二個角色
→ 重複直到所有角色完成
→ 標記今日每日正式任務已完成
→ 回角色管理等待
→ 等採集開始時間或下一個排程
```

---

### 12.2 每日補漏檢查

```text
到每日補漏檢查時間
→ 暫停採集
→ 回角色管理
→ 掃描角色
→ 依序登入所有角色
→ 檢查每日任務是否有漏做或漏領
→ 補做漏項
→ 執行主城資源與聯盟相關
→ 回角色管理
→ 標記今日補漏檢查已完成
```

---

### 12.3 每日任務限制

```text
每日正式任務每天只執行一次。
每日補漏檢查每天只執行一次。
信件清理類功能每天只執行一次，不因補漏重複亂刪。
每日任務優先權高於採集。
```

## 13. 異常與防卡

需要處理：

```text
角色被封視窗
網路斷線
卡載入
找不到出城
找不到放大鏡
找不到採集
找不到創建部隊
行軍失敗
部隊管理讀不到倒數
OCR 卡住
未知畫面
```

處理方式：

```text
每一步都有超時。
失敗先重試。
重試仍失敗，返回上一層或回角色管理。
仍無法恢復，發送通知並停止或等待人工處理。
```

## 15. 最終主流程總覽

```text
啟動腳本
→ 進入角色管理
→ 上下滑動兩次掃描角色
→ 記錄星標/指定角色
→ 顯示偵測到的角色倒數欄位

主循環：
    若到每日正式任務時間：
        依序登入所有角色
        執行每日正式任務
        回角色管理等待

    若到每日補漏檢查時間:
        依序登入所有角色
        執行每日補漏檢查
        回角色管理等待

    若啟用採集總控：
        若未啟用採集時間限制：
            根據角色倒數登入需要採集的角色

        若啟用採集時間限制，且目前在採集時間內：
            根據角色倒數登入需要採集的角色

        若啟用採集時間限制，且目前不在採集時間內：
            停在角色管理
            顯示：啟用採集時間未到，等待中
            不使用舊檔案倒數觸發登入

    每次登入角色後：
        判斷是否為綠色勾勾目前角色
        若是目前角色，不等待確定視窗，使用返回鍵回主城
        若不是目前角色，點確定登入
        登入完成後，以左上大頭貼或出城按鈕判斷主城

    採集角色流程：
        主城資源收取
        聯盟領土資源
        聯盟寶箱
        科技捐獻，只用藍色資源
        出城
        放大鏡
        選金/木/石
        最高等級往最低等級搜尋
        找到資源點
        採集
        創建部隊
        取消副將紅點
        行軍
        重複直到無法創建部隊
        部隊管理
        等部隊進入採集中
        讀最快部隊倒數
        回角色管理

    角色管理等待：
        顯示每個角色倒數或等待狀態
        等待下一個角色倒數、每日時間、採集開始時間
```

## 16. 必須特別注意的重點

1. 採集開始/結束時間必須有開關。
2. 未啟用採集時間限制時，不看開始/結束時間。
3. 採集時間未到時，顯示「啟用採集時間未到，等待中」。
4. 採集時間未到時，不可使用舊檔案倒數觸發採集。
5. 採集開始時間到後，重新掃描角色與建立本輪倒數。
6. 提前登入準備時間用秒數，不只用分鐘。
7. 角色管理上下滑動兩次，避免漏掃角色。
8. 滑動時不識別，停穩後才 OCR/找圖。
9. 綠色勾勾代表目前登入角色。
10. 點擊綠色勾勾角色時，不會出現確定/取消視窗。
11. 綠色勾勾角色要用返回鍵回主城，直到出現左上頭像或出城按鈕。
12. 採集停止條件以「是否出現創建部隊」為主，不以 5/5 為主。
13. 每個角色倒數使用最快完成部隊，不使用所有部隊時間總和。
14. 主城資源與聯盟相關不需要獨立分鐘間隔，每次角色登入後採集前固定執行。
15. 每日任務每天兩次：正式一次、補漏一次。
16. 每日正式完成後，若採集時間限制開啟，必須等採集開始時間才採集。
17. 角色登入最多重試 3 次。
18. 角色登入重試 3 次仍失敗，必須通知使用者。
19. 最後必須停在角色管理介面等待。
20. 採集選擇金 / 木 / 石時，必須套用 UI 的資源策略：平衡 / 隨機 / 固定金 / 固定木 / 固定石。
