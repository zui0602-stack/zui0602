# 萬龍覺醒自動化需求整理

這份資料夾是「工具無關」版本，目標是把目前所有需求整理成可放到 GitHub 的規格文件。

本整理不綁定任何特定腳本工具、語言或平台。後續可交給任何自動化軟體、AI 產生器、人工開發者或外包人員接續製作。

## 核心目標

製作一套萬龍覺醒自動化流程，主要處理：

- 多角色採集。
- 每日任務。
- 主城資源收取。
- 聯盟領土資源。
- 聯盟寶箱。
- 科技捐獻。
- 信件清理。
- 免費寶箱。
- 角色掃描。
- 部隊倒計時總控。
- 異常彈窗復位。
- 防卡通知。

最重要原則：

**每一個動作前後都必須偵測畫面，確認正確才點擊。畫面不對時不能硬點、不能跳步、不能亂跑下一步。**

## 開發方式建議

不要一開始就做成完整大流程直接跑。正確方式是：

1. 先完成畫面偵測框架。
2. 再完成每個入口的安全進入與返回。
3. 再做各功能小流程。
4. 最後才串成完整採集、每日任務、角色循環。

但是測試時可以集中測一批問題，不需要每修一個小點就反覆消耗使用量。

## 不可違反的規則

- 科技捐獻永遠不使用黃色寶石捐獻。
- 小紅點只代表要進去檢查，不代表可以亂點。
- 信件清理只在每日任務中每天一次。
- 聯盟相關不做總開關，固定包含領土、寶箱、科技捐獻。
- 採集與每日任務都要檢查主城產量資源與聯盟相關。
- 找不到目標畫面時，不允許繼續下一步。
- 卡住時要停止亂點，通知使用者處理。
