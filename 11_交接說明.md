# 11 交接給 AI 或開發者的說明

## 一、請先理解的重點

這不是單純連點腳本。

這是一套需要「每一步都偵測畫面」的遊戲自動化流程。若沒有畫面判斷與防亂點機制，任何彈窗、延遲、點錯、卡載入都可能讓整個流程亂掉。

## 二、交接時請明確告訴對方

請依照以下優先順序實作：

1. 畫面偵測框架。
2. 防亂點與返回復位。
3. 主城與野外切換。
4. 聯盟入口與聯盟頁內導航。
5. 科技捐獻安全判斷。
6. 採集派遣。
7. 每日任務。
8. 信件清理。
9. 角色掃描。
10. 倒計時總控。
11. 通知。

## 三、開發者必須遵守

必須遵守：

- 不綁定單一工具語言。
- 不依賴固定座標處理所有功能。
- 能用圖、文字、顏色確認就要確認。
- 點擊後必須確認下一畫面。
- 畫面錯誤時不能跳步。
- 高風險功能要更嚴格，例如科技捐獻。

## 四、最容易出錯的地方

最容易出錯：

- 聯盟入口點錯。
- 聯盟頁內找不到領土、寶箱、科技。
- 出城入口座標不準。
- 行軍按鈕裁到倒計時，導致找圖失敗。
- 科技捐獻誤點黃色寶石。
- 小紅點誤判造成亂點。
- 信件先刪除未領附件。
- 角色名稱 OCR 誤判導致登入錯角色。

## 五、建議實作架構

建議所有流程都拆成：

- `detectCurrentScreen`：偵測目前畫面。
- `waitForScreen`：等待指定畫面。
- `safeClick`：安全點擊。
- `recoverToKnownScreen`：返回復位。
- `handleUnknownScreen`：未知畫面處理。
- `notifyStuck`：卡住通知。

每個功能都不要直接點，要透過安全點擊與畫面確認。

## 六、測試方式

不要只測單一步，也不要每改一個小點就浪費完整測試。

建議：

1. 一次修一批明確問題。
2. 集中跑完整流程。
3. 把所有錯誤一次列出。
4. 再集中修下一輪。

但是每次完整測試前，要先確定：

- 入口能進。
- 返回能回。
- 不會點黃色寶石。
- 未知畫面不亂點。

## 七、目前需求狀態

需求已完整整理。

仍需實作或實測：

- 實際工具中的 API 對接。
- 素材裁切與命名。
- 聯盟頁內各入口定位。
- 出城入口定位。
- 科技捐獻藍色按鈕安全辨識。
- 角色掃描準確度。
- 通知方式。

## 八、給下一個 AI 的提示詞

可以直接貼：

```text
請閱讀本資料夾所有 Markdown 文件。
我要製作萬龍覺醒自動化流程，不綁定特定腳本語言。
重點是每一個動作都要偵測畫面、防止亂點。
科技捐獻永遠不能使用黃色寶石，只能使用藍色資源捐獻。
採集與每日任務都要檢查主城產量資源、聯盟領土、聯盟寶箱、科技捐獻。
請先根據文件整理實作架構，再依我指定的平台產生對應腳本。
```
