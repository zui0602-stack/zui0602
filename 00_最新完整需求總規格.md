## 1. 最終目標

製作一套《萬龍覺醒》自動精靈 `.zjs` 自動化總控流程。

核心架構：

```text
角色管理總控
→ 掃描角色
→ 統一登入角色
→ 登入後先處理主城資源與共用前置項目
→ 執行採集或每日任務
→ 讀取部隊倒數
→ 回到角色管理介面等待
→ 倒數或指定時間到後再登入對應角色
→ 持續循環
```

腳本最終不是單角色採集，而是完整多角色總控。

---

## 2. 最重要原則

1. 最終必須停留在「角色管理」介面等待。
2. 角色管理介面必須顯示目前已偵測角色的倒數或等待狀態。
3. 偵測到幾個角色，就顯示幾個角色，不固定顯示 5 個。
4. 所有登入方式都從「角色管理」統一進行。
5. 採集、每日正式任務、每日補漏檢查，都共用同一套角色登入流程。
6. 每一步動作前後都必須偵測畫面。
7. 畫面不對時不能硬點、不能跳步、不能亂跑下一步。
8. 所有等待必須有超時。
9. 所有步驟必須有成功判斷與失敗重試。
10. 圖片 / OCR 都要限定區域，不可長時間全屏 OCR。
11. 滑動期間不識別，滑動停穩後等待 500～1000ms 再 OCR / 找圖。
12. 不可以只靠固定延時。
13. 不可以做成一條超大流程，必須模組化。
14. 缺圖片素材時，要列出需要補的素材清單。
15. 卡住時停止亂點，通知使用者處理。
16. 每次重開腳本必須清空角色倒數快取，不可沿用舊倒數。
17. V1-C 採集停止條件是找不到「創建部隊」，不使用 5/5 OCR。

## 4. UI 介面需求

### 4.1 總開關

```text
[ ] 啟用採集總控
[ ] 啟用每日任務
[ ] 啟用每日補漏檢查
[ ] 啟用防卡通知
```

### 4.2 採集時間設定

採集時間必須有獨立開關，避免 UI 有時間欄位但使用者不想限制時間時，腳本誤判不採集。

```text
[ ] 啟用採集時間限制
採集開始時間：[00:00]
採集結束時間：[23:59]
```

規則：

```text
若「啟用採集時間限制」關閉：
    只要「啟用採集總控」開啟，就允許採集，不看開始/結束時間。

若「啟用採集時間限制」開啟：
    只有目前時間在採集開始～採集結束範圍內，才允許採集。

若開始時間小於結束時間：
    例如 08:00～23:59，代表同一天時間區間。

若開始時間大於結束時間：
    例如 18:50～07:59，代表跨日時間區間。

若時間格式錯誤：
    腳本不可亂跑，必須提示設定錯誤並停止或回角色管理等待。
```

### 4.3 採集時間外顯示規則

若「啟用採集時間限制」已開啟，且目前不在採集時間內：

```text
1. 腳本必須停在角色管理介面。
2. 不登入角色採集。
3. 不使用舊檔案紀錄的倒數時間去觸發採集。
4. 不因舊倒數到期而登入角色。
5. 角色管理等待畫面要顯示明確狀態：
   啟用採集時間未到，等待中
6. 若已掃描到角色，可顯示角色列表，但不得因舊倒數觸發登入。
7. 等待期間只允許檢查：
   - 每日正式任務時間是否到
   - 每日補漏檢查時間是否到
   - 採集開始時間是否到
8. 採集開始時間到後：
   - 重新掃描角色管理角色。
   - 重新建立本輪角色倒數狀態。
   - 不直接相信舊檔案倒數。
   - 再開始採集總控。
```

### 4.4 每日任務時間設定

每日任務每天只做兩次：

1. 正式執行一次。
2. 補漏檢查一次。

UI 需要包含：

```text
每日正式執行時間：[08:00]
每日補漏檢查時間：[20:00]
```

規則：

```text
每日正式任務：
    每天固定時間執行一次。
    每個角色都要跑一次。
    執行完成後，今日不再重複正式執行。

每日補漏檢查：
    每天固定時間執行一次。
    每個角色都要跑一次。
    主要檢查是否有每日任務漏做、漏領。
    補漏完成後，今日不再重複補漏檢查。

每日任務優先權高於採集。
每日任務時間到，採集必須暫停。
每日任務完成後，回角色管理等待。
若啟用採集時間限制，必須等到採集開始時間才開始採集。
```

### 4.5 提前登入準備時間

提前登入準備時間使用秒數，不只用分鐘。

```text
提前登入準備秒數：[180]
```

規則：

```text
角色倒數剩餘時間 <= 提前登入準備秒數：
    允許登入該角色準備採集。

若設定為 0：
    代表倒數到 0 才登入。
```

### 4.6 角色設定

```text
角色模式：
[ ] 自動偵測星標角色
[ ] 只登入指定角色

指定角色：
[ ] 角色 1
[ ] 角色 2
[ ] 角色 3
[ ] 角色 4
[ ] 角色 5
```

規則：

```text
最多支援 5 個角色。
角色數量不固定，可能 1～5 個。
偵測到幾個角色，就顯示幾個角色倒數。
被鎖角色、不需要登入的角色，不加星標。
```

### 4.7 採集設定

```text
資源策略：
    平衡
    隨機
    固定金
    固定木
    固定石

最高資源等級：[10]
最低資源等級：[1]
最大派遣嘗試：[10]
搜尋等待秒數：[3]
同資源同等級失敗次數：[2]
```

規則：

```text
資源類型只包含：
    金、木、石。

資源策略：
    平衡：盡量讓金木石輪流採集。
    隨機：每次隨機選金木石。
    固定金：只採金。
    固定木：只採木。
    固定石：只採石。

資源等級：
    從最高等級往最低等級搜尋。
    若最高等級找不到，就降一級繼續找。
    直到最低等級。
    最低仍找不到，再換下一種資源，並重設最高等級。
```

## 5. 角色管理掃描需求

### 5.1 掃描入口

腳本必須能進入：

```text
左上角頭像
→ 設定
→ 角色管理
```

或從其他畫面返回角色管理。

### 5.2 掃描方式

角色管理最多 5 個角色，但畫面可能無法一次看到全部，所以必須上下滑動掃描。

```text
進入角色管理後：
    先掃描當前畫面。
    向下滑動一次，等待畫面停穩，再掃描。
    再向下滑動第二次，等待畫面停穩，再掃描。
    再向上滑動一次，等待畫面停穩，再掃描。
    再向上滑動第二次，等待畫面停穩，再掃描。

總共上下滑動兩次，以免少掃角色。
滑動期間不做 OCR / 找圖。
每次滑動停止後等待 500～1000ms 再偵測。
```

### 5.3 掃描內容

需要偵測：

```text
角色名稱 / 角色 ID
星標狀態
綠色勾勾狀態
角色所在畫面位置
```

規則：

```text
星標角色：
    代表需要登入的角色。

指定角色：
    依 UI 勾選登入指定角色。

綠色勾勾：
    代表目前已登入角色。
```

### 5.4 去重規則

因為上下滑動會重複掃到同一個角色，所以必須去重。

去重依據優先順序：

```text
1. 角色 ID / 名稱 OCR 結果
2. 角色所在行特徵
3. 星標 + 名稱區域
```

同一角色不可重複加入角色清單。

## 6. 統一登入流程

所有功能都使用同一套登入流程，包括：

```text
採集
每日正式任務
每日補漏檢查
主城資源
聯盟相關
```

### 6.1 一般角色登入

流程：

```text
角色管理
→ 找到指定角色行
→ 點擊角色
→ 若出現「確定 / 取消」視窗
→ 點擊「確定」
→ 等待登入載入
→ 判斷登入完成
```

登入完成判斷：

```text
出現左上角大頭貼
或
出現「出城」按鈕
```

任一條件成立，即視為登入完成。

### 6.2 綠色勾勾角色登入特殊規則

角色管理介面上，如果某個角色行有綠色勾勾，代表目前已經登入該角色。

若目標角色行有綠色勾勾：

```text
點擊該角色或確認目前角色已登入。
不等待「確定 / 取消」視窗。
使用返回鍵返回。
重複返回，直到回到主城畫面。
以左上角大頭貼或出城按鈕判斷已回到主城。
```

### 6.3 登入失敗處理

登入失敗情況包括：

```text
找不到角色。
點擊角色無反應。
需要確認登入時，沒有出現「確定 / 取消」視窗。
非綠色勾勾角色登入後載入過久。
綠色勾勾角色按返回後，無法回到主城。
未出現左上角大頭貼。
未出現出城按鈕。
出現角色被封、斷線、異常彈窗。
畫面停在未知狀態。
```

處理方式：

```text
1. 單一角色登入最多重試 3 次。
2. 每次重試都要重新判斷目前畫面狀態。
3. 重試 3 次仍失敗：
   - 立即通知使用者。
   - 通知內容包含角色 ID / 角色名稱、失敗原因、目前畫面、重試次數。
   - 該角色標記為登入失敗。
   - 不再無限重試。
4. 若還有其他角色，可跳過該角色，繼續下一個角色。
5. 若所有角色都登入失敗，停在角色管理，顯示「所有角色登入失敗，等待人工處理」。
```

## 7. 採集流程

採集流程：

```text
登入角色
→ 回主城
→ 共用前置功能
→ 出城
→ 開放大鏡
→ 選資源
→ 選等級
→ 搜尋
→ 點資源點
→ 點採集
→ 判斷是否出現「創建部隊」
→ 有創建部隊：創建部隊 → 取消副將紅點 → 行軍 → 重新選資源/等級
→ 無創建部隊：結束 V1-C → 進入 V1-D 部隊管理
```

採集停止條件：

```text
點採集後找不到「創建部隊」
```

禁止：

```text
使用 5/5 OCR 當停止條件
使用右上角隊伍數 OCR 當停止條件
```

## 8. 部隊管理與倒數總控

部隊管理需要辨識：

```text
採集中
採集行軍中
返回中
```

規則：

```text
如果有「返回中」：
    等待或判斷是否可重新派遣。
    不直接記錄最終倒數。

如果有「採集行軍中」：
    等待行軍完成後再讀取採集中倒數。

如果全部都是「採集中」：
    讀取每支部隊倒數。
    取該角色最短正倒數。
    記錄到角色倒數表。
```

每次腳本重新啟動必須清空：

```text
角色時間[]
角色最短倒計時[]
角色下一次檢查時間[]
倒計時快取
部隊狀態暫存
目前角色索引暫存
每日任務本輪暫存
採集本輪嘗試次數
```

## 11. 每日任務模組設計

### 11.1 信件領取

```text
主城
→ 點信件
→ 優先找「一鍵已讀 / 領取」
→ 領取附件
→ 找「刪除已讀」
→ 返回主城
```

規則：

```text
先領附件，再刪除已讀。
找不到可領取按鈕就返回。
不得在信件頁無限重複點擊。
```

### 11.2 聯盟幫助

```text
主城
→ 找聯盟幫助 / 握手 / 幫助入口
→ 若出現「全部幫助」就點擊
→ 成功後返回主城
```

規則：

```text
幫助按鈕消失或提示已完成，就返回。
不可連續狂點。
```

### 11.3 英雄招募免費寶箱

影片中確認有：

```text
白銀寶箱
黃金寶箱
開啟
確定
再開一次
再開10次
全部開啟
```

規則：

```text
只處理免費次數或已有鑰匙可開啟的寶箱。
優先找「開啟」文字按鈕。
開啟後找「確定」返回。
如果顯示倒計時、今日已免費、沒有開啟，直接返回。
不主動消耗寶石。
```

### 11.4 神秘商人 / 黑市

規則：

```text
預設只檢查，不購買寶石商品。
只允許購買免費、資源貨幣、或使用者明確允許的商品。
遇到「售罄」直接跳過。
遇到紅色寶石價格，預設不買。
每個商品格最多判斷一次，避免重複購買。
```

### 11.5 任務列表 / 每日任務列表

規則：

```text
進入每日任務列表。
優先找「領取」。
若只有「前往」，先不自動亂跑，除非該任務已建立對應子流程。
任務列表完成後返回主城。
補漏檢查時再跑一次。
```

### 11.6 聯盟科技捐獻

規則：

```text
科技捐獻只用藍色資源。
永遠不使用黃色寶石。
如果只剩寶石捐獻，直接返回。
```

### 11.7 活動中心紅點與活動獎勵

規則：

```text
只處理有紅點、領取、開啟、免費、獎勵、通行幣類按鈕。
不亂點「前往」去打戰鬥或耗時活動。
遇到需要手動戰鬥、選路線、下一步決策的活動，先列為待確認。
```

### 11.8 聯盟領土收益 / 聯盟禮物 / 聯盟寶箱

規則：

```text
聯盟領土收益點擊領取後就返回。
因為領收按鈕可能不會消失，不能一直重複點。
聯盟寶箱以紅點或領取按鈕為準。
聯盟禮物領完返回。
```
