# 萬龍覺醒自動化需求整理

本倉庫是「萬龍覺醒自動化腳本」的最新需求規格入口。

## 請優先讀取

AI、Codex、開發者、外包人員開始前，請先讀：

```text
00_最新完整需求總規格.md
```

## 文件優先權

若 `00_最新完整需求總規格.md` 與其他舊文件、舊對話、舊規格有衝突，一律以：

```text
00_最新完整需求總規格.md
```

為準。

## 最新核心目標

製作一套萬龍覺醒自動化總控流程，包含：

- 多角色採集
- 角色管理掃描
- 統一登入流程
- 採集前主城資源收取
- 採集前聯盟領土 / 聯盟寶箱 / 科技捐獻
- 採集派遣
- 創建部隊
- 取消副將紅點
- 行軍
- 部隊管理倒數讀取
- 角色管理倒數等待總控
- 每日任務正式執行
- 每日補漏檢查
- 登入失敗通知
- 異常彈窗與防卡通知
- UI 開關與定時設定

## 最重要原則

腳本最終必須停留在「角色管理」介面等待。

角色管理介面需要顯示：

```text
啟用採集時間未到，等待中
```

或顯示目前已偵測角色的倒數狀態。

偵測到幾個角色，就顯示幾個角色，不固定顯示 5 個。

所有功能都必須以 `00_最新完整需求總規格.md` 為最新依據。
