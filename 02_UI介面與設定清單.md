# 02 UI 介面與設定清單

本文件整理需要給使用者操作的開關、時間、數值與角色設定。

## 一、執行模式

建議 UI：

```text
[ ] 啟用採集
採集開始時間：[00:00]
採集結束時間：[23:59]
[ ] 全天採集

[ ] 啟用每日任務
每日任務執行時間：[08:00]
[ ] 每日任務完成後，本日不再重複
```

規則：

- 每日任務優先權高於採集。
- 到每日任務時間時，先暫停採集，完成每日後再回採集。
- 全天採集勾選時，不看開始與結束時間。

## 二、角色登入方式

建議 UI：

```text
[ ] 自動偵測星標角色
[ ] 只登入指定角色

指定角色：
[ ] 角色 1
[ ] 角色 2
[ ] 角色 3
[ ] 角色 4
[ ] 角色 5
```

規則：

- 勾選自動偵測星標角色時，掃描角色管理內的星標角色。
- 勾選只登入指定角色時，只跑使用者勾選角色。
- 兩個都沒勾時，只跑目前登入角色。
- 如果兩個都勾，指定角色優先。

## 三、角色掃描設定

建議 UI：

```text
角色列表向下滑動次數：[2]
角色列表向上滑動次數：[2]
每次滑動後等待毫秒：[1000]
角色名稱去重：[開啟]
```

用途：

- 掃描角色管理列表。
- 找星標角色。
- 讀取角色名稱。
- 避免同一角色重複加入。

## 四、部隊倒計時總控

建議 UI：

```text
[ ] 啟用部隊倒計時總控
倒計時檢查間隔秒數：[60]
提前登入準備分鐘：[3]
[ ] 最短倒計時優先
```

用途：

- 定期掃描所有角色的部隊倒計時。
- 哪個角色最接近採集完成，就提前切到該角色準備。
- 避免資源採集完成後角色閒置太久。

## 五、採集設定

建議 UI：

```text
資源順序：[木, 石, 金]
最高資源等級：[10]
最低資源等級：[1]
搜尋等待秒數：[3]
同資源同等級失敗次數：[2]
最大派遣循環次數：[10]
```

規則：

- 資源順序可自訂。
- 從最高資源等級開始找。
- 同等級失敗達上限就降級。
- 最大派遣循環是防卡用，不代表部隊數量。

## 六、採集與每日共用檢查

這些功能不建議讓使用者關閉，因為是雙保險：

```text
[固定啟用] 主城產量資源收取
[固定啟用] 聯盟領土資源領取
[固定啟用] 聯盟寶箱領取
[固定啟用] 科技捐獻
```

規則：

- 採集前檢查一次。
- 每日任務時再檢查一次。
- 已領完或無可捐獻就跳過。
- 科技捐獻永遠只用藍色資源。

## 七、每日任務細項

建議 UI：

```text
[ ] 信件清理
[ ] 每日任務列表領取
[ ] VIP 領取
[ ] 寶箱領取
[ ] 特惠檢查
[ ] 每日寶物免費寶箱
[ ] 每日白銀免費寶箱
[ ] 每日黃金免費寶箱
[ ] 每日英雄免費寶箱
[ ] 每日一般英雄免費寶箱
```

預設建議：

- 信件清理：開。
- 每日任務列表領取：開。
- VIP：開。
- 已有素材的免費寶箱：開。
- 尚未確認細節的免費寶箱：先保留開關。

## 八、信件設定

建議 UI：

```text
[ ] 一鍵已讀/領取
[ ] 刪除已讀
[ ] 確認刪除已讀
```

規則：

- 一定先領附件再刪除。
- 不要先刪除未領附件。

## 九、異常與防卡設定

建議 UI：

```text
[ ] 啟用異常畫面返回復位
異常返回最大次數：[3]
返回後等待毫秒：[1000]

[ ] 啟用卡住監控
卡住超時分鐘：[5]

[ ] 啟用通知
通知方式：[待定]
通知對象：[待定]
```

規則：

- 畫面不對時先偵測，不立刻亂按。
- 超時才返回。
- 返回後再偵測。
- 多次失敗才通知。

## 十、日誌與測試設定

建議 UI：

```text
[ ] 顯示詳細日誌
[ ] 每個動作截圖記錄
[ ] 卡住時保存截圖
[ ] 測試模式，只執行單項功能
測試功能選擇：[主城資源 / 聯盟 / 出城 / 採集 / 每日 / 信件 / 科技捐獻]
```

測試模式非常重要，方便一次集中找出多個問題。
