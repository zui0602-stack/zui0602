# 03 動作偵測與防亂點規則

這份文件是整個專案最重要的規則。所有功能都必須遵守。

## 一、核心原則

任何動作都不能直接硬點。

每一步必須遵守：

```text
偵測目前畫面
確認是否為目標畫面
確認目標按鈕或入口存在
執行動作
等待畫面變化
確認是否進入下一個目標畫面
失敗則重試或復位
多次失敗則停止並通知
```

## 二、禁止行為

禁止：

- 不看畫面直接點固定座標。
- 畫面不對還繼續下一步。
- 找不到按鈕時猜位置亂點。
- 小紅點出現就直接點一堆位置。
- 用同一個座標處理所有彈窗。
- 沒有確認進入下一頁就開始下一步。
- 卡住後無限循環點擊。

## 三、每個動作的標準格式

每個動作都應該拆成：

1. 前置畫面檢查。
2. 目標元素檢查。
3. 點擊或操作。
4. 後置畫面檢查。
5. 失敗處理。

範例：

```text
動作：點擊採集

前置檢查：
- 目前必須在資源點面板。
- 畫面上必須有「採集」按鈕。

執行：
- 點擊「採集」。

後置檢查：
- 必須進入創建部隊或編隊相關畫面。

失敗處理：
- 如果沒進入下一頁，等待。
- 等待超時，返回上一頁。
- 重新偵測目前畫面。
```

## 四、畫面偵測層級

建議每個畫面至少用兩種訊號確認。

可用訊號：

- 畫面固定文字。
- 固定圖示。
- 固定按鈕。
- 顏色狀態。
- 小紅點。
- 位置區域。
- OCR 結果。

高風險畫面要多條件確認，例如科技捐獻：

- 必須有科技頁特徵。
- 必須有捐獻機會文字。
- 必須找到藍色捐獻按鈕。
- 不得出現黃色寶石捐獻點擊條件。

## 五、已知畫面清單

自動化流程至少要能辨識：

- 主城。
- 野外。
- 資源搜尋。
- 資源點面板。
- 採集面板。
- 創建部隊。
- 編隊。
- 行軍確認。
- 部隊管理。
- 角色管理。
- 角色登入確認。
- 每日任務。
- 信件。
- VIP。
- 免費寶箱。
- 特惠。
- 聯盟主頁。
- 聯盟領土。
- 聯盟寶箱或禮物。
- 科技捐獻。
- 載入中。
- 異常彈窗。
- 斷線或重連。

## 六、未知畫面處理

偵測到未知畫面時：

1. 不點擊任何功能按鈕。
2. 等待短時間重新辨識。
3. 如果仍未知，按一次返回。
4. 等畫面穩定。
5. 重新辨識已知畫面。
6. 如果回到已知畫面，接回流程。
7. 如果連續多次仍未知，停止流程並通知。

## 七、等待與超時規則

每個動作都要有等待與超時。

建議：

- 普通畫面切換：等待 1 到 3 秒。
- 載入畫面：等待 5 到 15 秒。
- OCR 搜尋文字：可重試 3 到 6 次。
- 找圖找色：可重試 3 到 5 次。
- 返回復位：最多 3 次。

超時後不要繼續下一步，要先進入失敗處理。

## 八、小紅點規則

小紅點只代表：

- 有可能有可領。
- 有可能有新內容。
- 需要進去檢查。

小紅點不代表：

- 一定能領。
- 可以直接點固定位置。
- 可以跳過畫面確認。

進入後仍要找：

- `一鍵領取`
- `領取`
- `收取`
- `開啟`
- `捐獻`

找不到就返回。

## 九、返回鍵復位規則

返回鍵只用於：

- 關閉異常彈窗。
- 從子頁回到上一層。
- 畫面不對時嘗試回正軌。

返回後必須：

- 等待畫面穩定。
- 重新偵測畫面。
- 不可按完返回就直接繼續原流程。

## 十、卡住判斷

以下情況判定卡住：

- 多次返回仍是未知畫面。
- 同一步驟連續超時。
- 找不到目標入口。
- 進入錯頁面且無法回主城或野外。
- 設備無反應。
- 遊戲斷線或卡載入。

卡住後：

- 停止點擊。
- 保存當前狀態。
- 記錄卡住原因。
- 通知使用者。
